# 🍜 好吃么 (HowTasty) - 去中心化美食点评 DApp

**好吃么** 是一个运行在 **Monad 测试网**上的去中心化投票与点评平台。

## 💡 项目愿景
在传统点评平台中，商家可以通过付费删评、雇佣水军等方式控制数据，导致消费者看到的评价真假难辨。
**好吃么** 旨在利用区块链的不可篡改性，打造一个“罗永浩式”的硬核点评空间。每一张选票、每一个评价都记录在 Monad 链上，公开透明，无法被中心化机构操控。

## 🚀 核心逻辑 (The Core Idea)

### 1. 商家录入：不仅仅是名字
为了防止数据混乱，商家录入采用 **“名称 + 模糊地址”** 的查重机制：
- **名称 (Name)**: 餐厅的品牌名。
- **坐标 (Location)**: 具体的街道或商圈。合约通过 `keccak256(name + location)` 确保在同一个地方不会有两个相同的商家。
- **录入者 (Curator)**: 每一个商家的首位录入者将永久记录在链上，作为该商家页面的“发起人”。

### 2. 硬核点评：证据即正义 (Proof of Taste)
为了打击水军，我们设计了不同的投票权重：
- ** 👍/👎 快速投票**: 适合路人，权重为 1。
- ** 📸 证据点评**: 必须包含 **IPFS 图片哈希**（如收银小票、真实的菜品照片）。这类点评在前端会有“已核实”标识，且对商家信誉分的影响权重为 5。

### 3. 去中心化共识：为什么选 Monad？
- **极速体验**: 点评应该是即时的。Monad 的高 TPS 让我们能像使用 Web2 应用一样快速完成链上交互。
- **透明性**: 任何中心化平台都可以后台改分，但在 Monad 链上，每一个赞或踩都是一笔不可撤回的交易。

### 4. 治理与防作恶
- **Sybil Attack 防御**: 考虑引入声望系统，或者要求账户有一定的交互历史才能进行高权重投票。
- **争议处理**: 未来可引入类似 Kleros 的去中心化仲裁机制，处理“虚假照片”等争议。

## � 代币经济与激励模型 (Tokenomics)

一个去中心化点评平台如果没有代币激励，很难在初期对抗 Web2 平台的流量优势。为了让“真相”具有经济动力，我们引入 **$TASTY** 代币和 **Reputation (声望)** 系统：

### 1. 角色与激励
- **录入者 (Curator)**: 首位录入该商家的用户。作为奖励，该商家后续所有点评产生的代币奖励中，有 **5%** 将作为“发现奖”分红给录入者。
- **点评者 (Reviewer)**: 
    - **普通点评**: 获得极少量奖励。
    - **硬核点评 (带图片证据)**: 获得大量 $TASTY 奖励。奖励金额受个人 **Reputation** 影响。
- **仲裁者 (Guardian)**: 质押 $TASTY 对疑似虚假点评发起挑战。挑战成功可获得被挑战者质押的代币。

### 2. $TASTY 代币循环
- **产出**: 100% 通过“点评挖矿”产生，鼓励真实数据上链。
- **消耗/销毁**:
    - 商家购买“首页曝光”（不影响评分，仅增加可见度）。
    - 发起仲裁挑战的门票。
    - 兑换社区合作餐厅的优惠券（未来愿景）。

### 3. 食客声望系统 (Reputation)
- 每一个钱包地址都有一个不可转让的声望值。
- **增加**: 点评被社区点赞、举报虚假信息成功。
- **减少**: 发布虚假点评被挑战成功、恶意举报。
- **核心逻辑**: 声望越高，说话越有份量（投票权重更高）。
